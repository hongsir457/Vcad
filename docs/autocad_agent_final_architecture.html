0. Implementation Mode

进入 Implementation Mode。不要做过渡版，不保留旧的正式执行入口。

禁止继续作为正式路径：
自然语言 → AgentLite → DSL / AutoLISP → 立即执行 → 结果卡片
正式路径：
NL → Intent → Task Plan → CAD-IR → Validator → Risk Policy（静态）→ Task Store → Preview / Dry Run → Risk Policy（动态复评）→ Confirm Token → Execute → Result → Audit Log
  • LLM 只负责理解、规划、生成结构化草案。
  • 所有安全裁决必须由服务端确定性策略完成。
  • 执行器只能消费已经落库、已确认（或低风险快速通道）、未过期、哈希一致的 CAD-IR。

1.2 v1.2 进一步修正（本版）

v1.1 遗留问题v1.2 处理方式
IR 内含 dry_run 字段，与"IR 落库后不可变"矛盾。preview 与 execute 共用同一份不可变 IR，该 flag 无法翻转。从 CAD-IR 移除 dry_run。执行模式属于 AdapterCommand 层：mode: "dry_run" | "execute"（见第 10 节）。
cad_result_v1 从未定义，但验收要求返回它。补充 4.4 节 Result Schema。
低风险任务在 API 层走不通：Risk 表说 Low 不需要确认，但 execute 强制要求 confirm_token。定义低风险快速通道：Low Risk 任务以 task_id + idempotency_key 直接 execute，免 confirm_token，但仍必须通过 Validator、静态/动态 Policy 与 Audit（见第 7 节）。
Risk Policy 时序矛盾：evaluateRisk 的入参 matched_count、layer_state、block_reference_count 在 Dry Run 之前并不存在。拆为两段式裁决：静态裁决（Dry Run 前，定确认等级与硬上限）+ 动态复评（Dry Run 后，只能维持或升级，不能降级）（见第 6 节）。
AutoLISP *error* 兜底只关闭 undo 组、不回滚，与高风险 all_or_nothing 语义冲突——出错时半成品修改会保留。all_or_nothing 模式下，兜底必须在 UNDO _END 之后立即整组回退（见第 10 节修正后的模板）。
状态机缺 rejected 终态，Validator/Policy 的"拒绝"无处可去；needs_clarification 被画成主链直线。状态机改为分支图，补充 rejected 终态（见第 9 节）。
Preview 响应缺前端展示字段（风险等级、二次确认、新值、可撤销、过期时间）。cad_preview_v1 补齐展示字段，且全部来自服务端裁决结果（见第 8 节）。
document_id 出现在 LLM 生成的 Intent 中，LLM 无法得知真实文档身份。document_id 由插件端会话上下文注入，LLM 输出中不含该字段，服务端拼装（见 4.1 节）。
次要：状态机有 cancelled 但无取消接口；export_objects 未限制导出目录。新增 POST /cad-agent/cancel；export_objects 仅允许写入受控导出目录。

2. 最终主流程

用户自然语言
  ↓
Intent Parser
  ↓
Clarification Gate（缺范围/缺对象/歧义时进入澄清，补充后重回 Intent Parser）
  ↓
Task Planner
  ↓
CAD-IR Generator（服务端注入 document_id 等会话上下文）
  ↓
CAD-IR Validator ──不通过──→ rejected（终态，落审计）
  ↓
Risk Policy 静态裁决（定风险下限、确认等级、对象数硬上限）
  ↓
Task Store（task_id + immutable IR + ir_hash + server_safety）
  ↓
├─ Low Risk：动态复评通过后直接 Execute（快速通道，仍走 Adapter/Executor/Audit）
└─ Medium / High：
     Preview / Dry Run
       ↓
     Risk Policy 动态复评（matched_count / layer_state / 引用数；只升不降）──超限──→ rejected
       ↓
     Confirm / Second Confirm（签发限时 confirm_token）
       ↓
     Execute 前复核 Dry Run（TOCTOU 检查，不一致 → stale）
       ↓
     Execution Adapter（AutoLISP / .NET，mode=execute）
       ↓
     AutoCAD Executor（幂等执行 + Undo Group + 兜底回滚）
       ↓
     Result Reporter
       ↓
     Audit Log

4. Schema 与核心数据结构

4.1 Intent

{
  "schema": "cad_intent_v1",
  "intent": "modify_objects",
  "target": {
    "space": "model",
    "layer": "C-ANNO",
    "entity_type": "CIRCLE",
    "selection_scope": "layer"
  },
  "operation": {

document_id 来源：LLM 无法得知真实文档身份，Intent 中不包含 document_id。文档绑定由插件端会话上下文提供，CAD-IR Generator 在服务端拼装时注入；执行时 Executor 校验活动文档与绑定一致，不一致则中止。

5. Action 白名单与禁用项

Action风险说明
query_objectsLow查询对象，不改图。
count_objectsLow统计对象数量。
highlight_objectsLow高亮/选择，不改图。
export_objectsLow导出统计或对象信息，仅允许写入受控导出目录，路径不接受用户/LLM 自由输入。
set_propertyMedium统一表达文字高度、颜色、线型、半径、标注样式、图层属性等属性类修改。作用于图层/样式等共享对象时动态升级为 High。
move_objectsMedium移动对象，必须 preview + confirm。
copy_objectsMedium复制对象，必须 preview + confirm。
delete_objectsHigh删除对象，必须二次确认。
explode_blocksHigh炸开块，必须二次确认。
purge_layersHigh清理图层，必须二次确认。
modify_block_definitionHigh修改块定义，必须显示引用实例数并二次确认。
scale_objectsHigh缩放对象，正式入白名单，高风险。

v1 禁用项

  • global_replace：v1 禁用。Validator 直接拒绝，不得进入 Adapter。
  • modify_xref：v1 禁用。Validator 直接拒绝，不得进入 Adapter。
  • 自由脚本执行：禁用。不得作为正式 API 暴露。

6. 服务端 Risk Policy（两段式）v1.2 修正

Risk Policy 是唯一安全裁决来源，不得相信 LLM 写入 IR 的风险字段。v1.1 的单次 evaluateRisk 依赖 matched_count 等运行时数据，但这些数据在 Dry Run 之前不存在——因此裁决拆为两段：

第一段：静态裁决（policy_applied，Dry Run 前）

static_safety = evaluateStaticRisk({
  action,                  // 白名单内动作 → 基础风险等级
  selector.selection_scope,
  property.name,           // 共享对象属性（图层/样式）→ 升级
  document_binding         // 缺 document 绑定 → 拒绝
})
// 产出：risk_level 下限、requires_confirmation、
//       requires_second_confirmation、max_affected_objects 硬上限、failure_mode
// 写入 Task Record.server_safety

第二段：动态复评（Dry Run 完成后 / Execute 复核时）

dynamic_safety = reviewWithRuntime(static_safety, {
  matched_count,           // 超硬上限 → rejected
  layer_state,             // locked/frozen/off → 阻断对应对象
  block_reference_count,   // 块引用数 → 展示并维持 High
  document_revision
})
// 约束：动态复评只能维持或升级风险等级，不能降级；
//       任何降级路径都视为实现错误。

硬规则

  • 无 document 绑定：拒绝。
  • 无明确 selector 且是修改类 action：拒绝。
  • selection_scope = global：至少高风险并要求二次确认。
  • 命中数超过策略硬上限：拒绝或要求缩小范围，不能使用 LLM 上报的上限绕过（取 IR 声明值与硬上限的较小者）。
  • 删除、炸开、清理、修改块定义、缩放：高风险。
  • 图层 locked/frozen/off：默认阻断对应对象修改，并返回 blocked_count。

策略上限示例

风险默认对象上限确认要求
Low10000免确认（快速通道，见第 7 节），但必须经过 CAD-IR + Validator + 两段裁决 + Audit。
Medium200Preview + Confirm。
High50Preview + Second Confirm。

7. Task Store、Confirm Token 与幂等

所有 preview、confirm、execute 均按 task_id 引用不可变 Task Record，禁止客户端重新提交 IR 执行。

接口

POST /cad-agent/interpret     → 返回 intent、plan、task_id（或澄清请求）
POST /cad-agent/preview       → 入参 task_id，只 Dry Run
POST /cad-agent/confirm       → 入参 task_id，签发 confirm_token
POST /cad-agent/execute       → 入参 task_id + confirm_token + idempotency_key
POST /cad-agent/cancel        → 入参 task_id，置 cancelled 终态
GET  /cad-agent/tasks/:task_id

低风险快速通道 v1.2 补充

  • Low Risk 任务（server_safety 裁决为 Low）允许 execute 仅携带 task_id + idempotency_key，免 confirm_token。
  • 前提：状态为 policy_applied，且执行前的动态复评通过（命中数在 Low 上限内）。
  • 快速通道仍然全程经过 CAD-IR → Validator → Adapter → Executor → Audit，只是跳过 Preview/Confirm 交互。
  • Medium / High 任务的 execute 缺 confirm_token 一律拒绝。

Confirm Token

  • 绑定 task_id、ir_hash、document_id、preview_hash。
  • 设置有效期，例如 2–5 分钟。
  • Medium 风险使用一级确认；High 风险使用二级确认（第二次独立 confirm 调用）。
  • 过期、哈希不一致、状态不一致时拒绝执行。

幂等

  • execute 必须要求 idempotency_key。
  • 同一 task_id + idempotency_key 重复提交，只返回第一次执行结果。
  • 任务进入 terminal state 后不得重复执行——即使更换 idempotency_key 也必须拒绝。

8. Preview / Dry Run / TOCTOU

中高风险任务必须先 Dry Run。Dry Run 不改图，只返回影响范围、对象数量、图层状态、可修改对象数量和快照摘要。Dry Run 复用同一条 Adapter → Executor 链路（mode = dry_run，只生成选择/统计命令），保证 Preview 数字与真实执行使用同一选择逻辑。

{
  "schema": "cad_preview_v1",
  "task_id": "cad_task_20260611_0001",
  "status": "preview_ready",
  "preview_hash": "sha256:...",
  "document_revision": "rev_103",
  "risk_level": "medium",
  "requires_confirmation": true,
  "requires_second_confirmation": false,
  "expires_at": "2026-06-11T00:15:00Z",
  "operation": {
    "action": "set_property",
    "property": "radius",
    "new_value": 5.0
  },
  "impact": {
    "matched_count": 128,
    "modifiable_count": 126,
    "blocked_count": 2,
    "object_set_hash": "sha256:..."
  },
  "undo": {
    "available": true,
    "will_create_group": true
  },
  "warnings": [
    { "code": "LAYER_LOCKED_OBJECTS", "message": "2 个对象所在图层不可修改" }
  ]
}
risk_level / requires_* 来自 Task Record 的 server_safety（动态复评后），不是 IR 的 advisory_safety。Preview Card 展示所需的全部字段（风险、新值、可撤销、过期时间）必须由本响应携带，前端不得回读 IR。v1.2 补充

Execute 前复核

  • execute 前必须再次 Dry Run。
  • 对比 matched_count、object_set_hash、document_revision。
  • 不一致则中止，任务状态置为 stale，要求用户重新 preview。

9. 任务状态机与失败语义

状态机 v1.2 修正：分支化 + rejected 终态

created → interpreted ─┬─→ needs_clarification → interpreted（用户补充后）
                       └─→ ir_validated ─┬─→ rejected      （Validator / 静态 Policy 拒绝，终态）
                                         └─→ policy_applied

policy_applied ─┬─→ executing                               （Low Risk 快速通道，动态复评通过）
                └─→ preview_ready ─┬─→ rejected             （动态复评超限，终态）
                                   └─→ confirmed → [second_confirmed] → executing

executing → succeeded | partial_succeeded | failed

preview_ready / confirmed / second_confirmed
  → expired   （确认超时，终态）
  → stale     （执行前复核发现图纸已变化，终态）
  → cancelled （用户取消，终态）
  • 终态：succeeded / partial_succeeded / failed / rejected / expired / stale / cancelled。终态任务不可复用，需重新 interpret。
  • 每次状态迁移写入 Audit Log。

失败语义

模式适用行为
all_or_nothing删除、缩放、块定义修改等高风险动作任何对象失败则整组回滚，结果为 failed。
partial_allowed属性类修改允许修改可修改对象，失败对象进入 errors，结果为 partial_succeeded。

默认策略：High Risk 使用 all_or_nothing，Medium Risk 可按 action 配置。failure_mode 由静态裁决写入 server_safety，不由 LLM 指定。

10. Adapter / Executor / AutoLISP 兜底

Adapter 规则

  • Adapter 只能消费 Task Store 中已校验的 CAD-IR。
  • Adapter 不接收自然语言、不接收 Task Plan、不接收 LLM 生成脚本。
  • Adapter 对同一 IR + 同一 mode 的输出必须确定（同输入同输出），便于审计与测试。
  • IR 中的字符串值（图层名等）写入 LISP 前必须转义/参数化，防止注入。
  • Adapter 输出 AdapterCommand，由 Executor 执行。
{
  "schema": "adapter_command_v1",
  "adapter": "autocad_lisp",
  "task_id": "cad_task_20260611_0001",
  "ir_hash": "sha256:...",
  "command_type": "lisp",
  "mode": "execute",                 // "dry_run" | "execute"  ← v1.2：模式在此层，不在 IR
  "command": "<generated_lisp>",
  "safe_to_execute": true
}
  • mode = dry_run 时只允许生成选择/统计命令（ssget 级别），禁止出现任何修改实体的调用，且不创建 undo group。

AutoLISP 错误兜底 v1.2 修正：关组后必须回滚

v1.1 模板的 *error* 只执行 UNDO _END，半成品修改会保留在图纸上，与 all_or_nothing 冲突。修正后：

(defun c:CAD_AGENT_EXEC (/ olderr undo_open)
  (setq olderr *error*)
  (defun *error* (msg)
    ;; 兜底顺序：先关闭未结束的 undo 组，再立即整组回退，
    ;; 保证 all_or_nothing 语义下不留下半成品修改
    (if undo_open
      (progn
        (command "_.UNDO" "_END")
        (command "_.U")))
    (setq *error* olderr)
    ;; return structured error payload here
    (princ)
  )
  (command "_.UNDO" "_BEGIN")
  (setq undo_open T)
  ;; generated deterministic operations from CAD-IR only
  (command "_.UNDO" "_END")
  (setq undo_open nil)
  (setq *error* olderr)
  ;; return structured success payload here
  (princ)
)
  • partial_allowed 模式下，单对象失败在脚本内部逐对象捕获并计入 errors，不触发 *error*；*error* 只兜底不可恢复的脚本级异常，此时同样整组回退并返回 failed。
  • 结构化结果（成功/失败对象句柄、计数）必须由脚本返回，供 execution_result_parser 解析。

Executor

  • Executor 只接收 AdapterCommand。
  • 执行前检查 task state、confirm_token（Low Risk 快速通道除外）、ir_hash、idempotency_key。
  • 执行前核对活动文档与 Task Record 的 document_id 一致，已切换则中止。
  • 同一文档同一时间只允许一个修改类任务在执行（串行）。
  • 执行后必须解析结构化结果并写入 Task Store 与 Audit Log。

11. 前端卡片

前端必须从单一结果卡片升级为任务流卡片。

  • Intent Card：展示理解结果和可能歧义。
  • Clarification Card：缺范围、缺对象、缺图层时要求用户补充。
  • Plan Card：展示任务步骤。
  • Preview Card：展示影响范围、风险等级、命中数、不可修改对象、新值、可撤销状态、确认截止时间（全部取自 cad_preview_v1）。
  • Confirm Card：一级确认或二级确认；按钮防重复点击；token 过期时提示重新 Preview。
  • Result Card：展示结构化执行结果（cad_result_v1）。
  • Error / Stale Card：展示拒绝、过期、图纸变化、确认失效、执行失败等原因（错误码 + 本地化文案）。

12. Audit Log

Audit Log 是 P0，不是预留项。每个关键状态必须写日志。

{
  "schema": "cad_audit_log_v1",
  "event_id": "evt_001",
  "task_id": "cad_task_20260611_0001",
  "event_type": "execute_succeeded",
  "document_id": "doc_001",
  "ir_hash": "sha256:...",
  "actor": "user",
  "timestamp": "2026-06-11T00:00:00Z",
  "payload": {
    "matched_count": 128,
    "modified_count": 126,
    "failed_count": 2
  }
}

必须记录：interpret、validate、rejected、policy_applied、dynamic_review、preview、confirm、second_confirm、execute_start、execute_succeeded、execute_failed、stale、expired、cancelled。

13. 本轮范围

必须完成

  • Intent / Task Plan / CAD-IR / Validator。
  • Action 白名单 v1.2。
  • 服务端 Risk Policy 两段式裁决（静态 + 动态复评，只升不降）。
  • Task Store：task_id、ir_hash、不可变 IR、状态机（含 rejected 终态）。
  • Preview / Dry Run / TOCTOU stale check（Preview 响应含全部展示字段）。
  • Confirm Token、防篡改、防重放、幂等 execute、低风险快速通道、cancel 接口。
  • AutoLISP Adapter（含 mode 区分、注入防护）与 .NET Adapter 接口预留。
  • AutoLISP undo group + *error* 兜底（关组后整组回退）。
  • Result Schema（cad_result_v1）、Result Reporter、Audit Log。
  • 前端 Intent / Plan / Clarification / Preview / Confirm / Result / Error 卡片。
  • 旧 immediate execute 路径隔离或禁用。

本轮不做

  • 云端 CAD。
  • 语音识别。
  • 多 CAD 平台完整适配。
  • 构件语义识别。
  • 工程量计算。
  • 复杂图纸理解。
  • 批处理队列。
  • 多 Agent 协同。
  • global_replace / modify_xref（Validator 直接拒绝）。

14. 验收标准

  1. 自然语言不能直接进入执行器。
  2. LLM 不能直接输出 AutoLISP 并立即执行。
  3. 所有执行必须经过 Intent → Task Plan → CAD-IR → Validator。
  4. Action 白名单与 Risk Policy 一致，不存在 Safety 表可引用但 Validator 拒绝的 action。
  5. modify_text_height 等属性修改必须通过 set_property 表达。
  6. scale_objects 在白名单内且为高风险。
  7. global_replace、modify_xref 在 v1 禁用，进入系统时必须拒绝。
  8. Risk Policy 由服务端两段式裁决：静态裁决在 Dry Run 前完成，动态复评只能维持或升级风险，篡改 IR 中 advisory_safety 无法降低安全要求（需有测试用例）。
  9. CAD-IR 不包含执行模式字段；dry_run 与 execute 由 AdapterCommand 的 mode 区分，IR 落库后不可变（需有测试用例）。
  10. Task Store 中 IR 落库后不可变，execute 只能按 task_id 引用。
  11. Confirm Token 必须绑定 task_id、ir_hash、preview_hash、document_id，且有有效期。
  12. Low Risk 任务可经快速通道免确认执行，但必须通过 Validator、两段裁决与 Audit；Medium/High 缺 confirm_token 一律拒绝（需有测试用例）。
  13. Execute 必须幂等，重复点击不能重复修改图纸；终态任务更换 idempotency_key 也不得重新执行。
  14. Execute 前必须重新 Dry Run，比对 preview 快照，不一致则 stale 中止。
  15. 模糊输入必须进入 Clarification，不允许硬猜后改图。
  16. 中风险操作必须 Preview + Confirm；高风险操作必须 Preview + Second Confirm。
  17. 没有明确范围时，禁止修改图纸。
  18. 删除 / 炸开 / 清理 / 修改块定义 / 缩放禁止直接执行。
  19. 执行前必须创建 undo group；AutoLISP *error* 兜底必须在关闭 undo group 后整组回退，出错时图纸不得残留半成品修改（需有测试用例）。
  20. document_id 由插件端会话注入，LLM 输出不含该字段；执行前必须核对活动文档一致。
  21. 执行后必须返回 cad_result_v1（4.4 节定义），包含命中数、成功数、失败数、耗时、撤销状态。
  22. 状态机包含 rejected 终态，Validator / Policy 拒绝必须落审计。
  23. Audit Log 必须记录关键状态，不得作为后续预留。
  24. 旧 immediate execute 路径必须隔离或禁用，不接入正式 UI/API。
