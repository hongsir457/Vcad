AutoCAD 自然语言 Agent 最终架构执行口径 v1.3

用途：发给开发 Agent 的实现口径。v1.2 修复了 IR 不可变、confirm、risk、result、TOCTOU、状态机、preview 字段等落地问题；v1.3 修复 v1.2 遗留的 AutoLISP 回滚错误（UNDO BACK / command-s）、failure_policy 信任 LLM、expired 终态缺失、partial 结果语义、命名不一致等问题。

0. v1.3 硬性修正（在 v1.2 基础上）

  1. AutoLISP 兜底禁止使用 UNDO _BACK。UNDO Back 回退到上一个 undo 标记，无标记时会回退整个会话，比不回滚更危险。正确做法：UNDO _END 关组后，该组已塌缩为一步，用 _.U 回退这一步（第 16 节）。
  2. 错误处理器内必须使用 command-s。AutoCAD 在 *error* 上下文中限制 command 调用，兜底回滚必须用 command-s，否则在最需要时可能失败。开组使用标准选项 _Begin（非 _GROUP）。错误消息写入 JSON 前必须转义。
  3. failure_policy 不得信任 LLM。IR 中只允许 failure_policy_request；生效值由 Static Risk Policy 钳制写入 Task Record：高风险强制 all_or_nothing（不是"默认"，不可被 IR 覆盖）。
  4. 状态机补 expired 终态、补 partial_succeeded。确认过期有去处；best_effort 部分成功有对应结果状态。状态机改为分支图，rejected / stale 不再排在主链上。
  5. 统一命名。结果状态与状态机一致使用 succeeded / partial_succeeded / failed；preview 的 blocked 与 result 的对象分类统一为 blocked（废弃 result 中的 skipped 叫法）。
  6. cancel 适用于任何非终态。不再限于 pending / preview_ready / waiting_confirmation；needs_clarification、confirmed 等同样可取消。
  7. confirm_token 增绑 preview_snapshot_hash。用户确认的是"看到的那次 preview"；second-confirm 必须先持有有效的一级 confirm_token。
  8. Low 风险的动态复评时点明确为 execute preflight。低风险不调 preview，运行时数据来自 execute 的 preflight Dry Run，复评在该步完成。
  9. drawing_version / drawing_fingerprint 来源落地。DWG 无内置版本号，由插件端数据库修改反应器（或命令计数器）维护；机制不可用时 stale 判定退化为仅比对 preview 快照各计数与对象集合摘要，并写 Audit。

v1.2 修正项（全部保留）

IR 不含运行模式字段（mode 属于 AdapterCommand）；补齐 cad_result_v1；Low 风险免 confirm_token 执行；Risk Policy 两段裁决（动态只升不降）；execute 前 preflight Dry Run 防 TOCTOU；状态机含 rejected / stale / cancelled；Preview 响应补齐前端字段；document_id 由插件会话注入；export_objects 限制导出目录。

3. Session Context：文档身份由插件端注入

LLM 不得生成真实 document_id、drawing_version、active_layout。这些字段由 AutoCAD 插件端会话注入。

{
  "schema": "cad_session_context_v1",
  "session_id": "sess_20260611_001",
  "document_id": "doc_autocad_8f3a",
  "drawing_fingerprint": "sha256:...",
  "drawing_version": 42,
  "active_space": "model",
  "active_layout": "Model",
  "user_id": "user_001",
  "allowed_export_roots": [
    "C:/Users/Public/Documents/SmartCAD/Exports",
    "%TEMP%/SmartCAD/Exports"
  ]
}

drawing_version / drawing_fingerprint 的来源（v1.3）：DWG 没有内置修订号。drawing_version 由插件端 drawing_revision_tracker 通过数据库修改反应器（或命令结束计数器）维护，文档打开时从当前会话起算、单调递增。drawing_fingerprint 不要求全文件哈希（开销过大），允许实现为「版本号 + 保存状态 + 对象总数」的轻量摘要。追踪机制不可用（反应器注册失败）时，stale 判定退化为仅比对 preview 快照中的各项计数与对象集合摘要，并向 Audit Log 写 revision_tracking_degraded 事件。

7. CAD-IR：不可变、无 dry_run 字段

CAD-IR 是执行层唯一业务输入。IR 一旦进入 Task Store 即不可变。

{
  "schema": "cad_ir_v1",
  "action": "set_property",
  "selector": {
    "space": "model",
    "layout": "Model",
    "layer": "C-ANNO",
    "entity_type": "CIRCLE",
    "selection_scope": "layer"
  },
  "operation": {
    "property": {
      "name": "radius",
      "value": 5.0,
      "unit": "drawing_unit"
    }
  },
  "execution_constraints": {
    "undo_group": true,
    "max_affected_objects_request": 500,
    "failure_policy_request": "all_or_nothing"
  },
  "advisory_safety": {
    "llm_risk_hint": "medium",
    "llm_requires_confirmation_hint": true
  },
  "reporting": {
    "return_object_ids": true,
    "return_failed_objects": true
  }
}

禁止：

  • IR 中出现 dry_run、execute 等运行模式字段。
  • IR 中出现 AutoLISP / C# / 自由脚本文本。
  • 信任 IR 中由 LLM 给出的风险字段。
  • v1.3：信任 IR 中的 failure_policy_request。该字段仅是请求，生效值由 Static Risk Policy 钳制写入 Task Record；高风险强制 all_or_nothing，IR 无法覆盖。

9. CAD-IR Validator

  1. schema 必须是 cad_ir_v1。
  2. action 必须属于白名单，且不是 disabled_v1。
  3. selector 必须明确 space/layout/entity_type/layer 或当前选择集。
  4. 修改类操作禁止无范围全图修改。
  5. set_property 的 property.name 必须属于实体类型允许属性。
  6. property.value 类型和单位必须合法。
  7. execution_constraints.undo_group 对修改类必须为 true。
  8. failure_policy_request 只能是 all_or_nothing 或 best_effort。Validator 只校验取值合法；生效值由 Static Risk Policy 决定：高风险一律强制 all_or_nothing，Medium 取请求值与策略允许值的交集。
  9. export_objects 必须指定导出类型，最终路径必须落在 allowed_export_roots 内。
  10. IR 任何字符串字段不得包含可执行脚本/LISP 片段（模式拦截）。

Validator 拒绝时，任务进入 rejected 终态，并写 Audit Log。

10. Risk Policy：两段裁决

10.1 静态裁决 Static Risk Policy

输入只使用 IR 中已经存在、无需访问 AutoCAD 运行态的数据。

{
  "schema": "cad_static_risk_v1",
  "inputs": {
    "action": "set_property",
    "selection_scope": "layer",
    "entity_type": "CIRCLE",
    "property_name": "radius",
    "requested_max_affected_objects": 500,
    "requested_failure_policy": "all_or_nothing"
  },
  "decision": {
    "risk_level": "medium",
    "confirmation_level": "confirm_required",
    "policy_max_affected_objects": 500,
    "effective_max_affected_objects": 500,
    "effective_failure_policy": "all_or_nothing"
  }
}

effective_max_affected_objects = min(requested, policy)；effective_failure_policy 由策略按风险等级裁定（High 强制 all_or_nothing），二者写入 Task Record，执行层只读 Task Record，不读 IR 请求值。

10.2 动态复评 Dynamic Risk Re-evaluation

拿到 matched_count、layer_state、block_reference_count、document_version 等运行态数据后，再动态复评。动态复评只能升级风险，不能降级。

时点（v1.3 明确）：Medium / High 在 Preview 的 Dry Run 之后复评；Low 风险不调 preview，复评在 execute 的 preflight Dry Run 中完成——复评若升级风险，则中止快速通道执行，任务转入 preview_ready，按升级后的等级走确认流程。

{
  "schema": "cad_dynamic_risk_v1",
  "static_risk_level": "medium",
  "runtime_inputs": {
    "matched_count": 128,
    "modifiable_count": 126,
    "blocked_count": 2,
    "layer_state": { "locked": false, "frozen": false, "off": false },
    "block_reference_count": null,
    "document_version": 42
  },
  "decision": {
    "risk_level": "medium",
    "confirmation_level": "confirm_required",
    "requires_second_confirmation": false
  }
}
等级Action处理方式
Lowquery_objects、count_objects、highlight_objects、export_objects必须经过 CAD-IR，可直接 execute；不需要 confirm_token；动态复评在 preflight 完成
Mediumset_property、move_objects、copy_objectsPreview + Confirm
Highdelete_objects、explode_blocks、purge_layers、modify_block_definition、scale_objectsPreview + Second Confirm；failure_policy 强制 all_or_nothing
Rejectedglobal_replace、modify_xref、无明确范围修改、非法导出路径拒绝执行

12.1 execute 请求

{
  "task_id": "task_20260611_0001",
  "idempotency_key": "client-generated-uuid",
  "confirm_token": "required_for_medium_and_high",
  "second_confirm_token": "required_for_high"
}
  • Low 风险：task_id + idempotency_key 可直接 execute（preflight 动态复评通过为前提）。
  • Medium 风险：必须提供有效 confirm_token。
  • High 风险：必须提供有效 confirm_token + second_confirm_token。
  • execute 必须幂等：同一 task_id + idempotency_key 重复提交，只返回首次结果，不重复执行。

14. Preview/Execute TOCTOU 防护

execute 前必须重新 Dry Run（preflight）。Medium / High 任务与 preview_snapshot 比对（下列各条）；Low 风险任务没有 preview 快照，preflight 只做动态复评（第 10.2 节），不做快照比对。

  1. drawing_fingerprint 或 drawing_version 变更：中止，任务置为 stale。
  2. matched_count、modifiable_count、blocked_count 变更：中止，任务置为 stale。
  3. 图层 locked/frozen/off 状态变更：中止，任务置为 stale。
  4. 块引用数变更：中止，任务置为 stale。
  5. 修订追踪机制退化时（第 3 节），按第 2–4 条比对，并已在 Audit 中记录退化事件。

stale 是终态，旧确认与旧 token 全部作废。前端 Stale Card 的"重新预览"按钮由服务端基于原 IR 重建新 task（新 task_id，重新走 Validator → Policy → Preview），不复用旧任务记录。

16. AutoLISP Undo 与错误兜底（v1.3 修正）

v1.2 模板有两处会造成事故的错误，禁止沿用：

  1. *error* 内使用 (command ...)：AutoCAD 在错误回调上下文中限制 command，兜底可能根本不执行。必须使用 command-s。
  2. 回滚使用 UNDO _BACK：Back 回退到上一个 undo 标记，无标记时回退整个会话，会摧毁用户当天的全部工作。关组后该组已塌缩为一步，正确回滚是 _.U（撤销一步）。
(defun smartcad-command (/ olderr)
  (setq olderr *error*)
  (defun *error* (msg)
    ;; 兜底：错误处理器上下文必须用 command-s，不得用 command。
    ;; 先关闭未结束的 undo 组（组塌缩为一步），再用 _.U 仅回退这一步，
    ;; 满足 all_or_nothing；禁止 UNDO _BACK（无标记时回退整个会话）。
    (if smartcad-undo-open
      (progn
        (command-s "_.UNDO" "_END")
        (setq smartcad-undo-open nil)
        (command-s "_.U")
      )
    )
    (setq *error* olderr)
    ;; msg 必须经 JSON 转义后写入，禁止裸拼接（引号/反斜杠会产出非法 JSON）
    (smartcad-return-json
      (smartcad-build-error-json "EXECUTION_FAILED_ROLLED_BACK" msg))
    (princ)
  )

  (command "_.UNDO" "_Begin")
  (setq smartcad-undo-open T)

  ;; execute modifications here
  ;; best_effort 模式：单对象失败用 vl-catch-all-apply 逐对象捕获并计入
  ;; errors，不触发 *error*；*error* 只兜底不可恢复的脚本级异常。

  (command "_.UNDO" "_END")
  (setq smartcad-undo-open nil)
  (setq *error* olderr)
  (smartcad-return-json (smartcad-build-success-json))
  (princ)
)
  • 开组使用标准选项 _Begin。
  • smartcad-build-error-json 负责对 msg 做 JSON 转义；结构化结果（成功/失败对象句柄、计数）由脚本返回，供 Executor 解析。
  • .NET Adapter 后续以 Transaction.Abort 实现同等回滚语义，块定义修改等高风险动作建议优先迁移到 .NET 事务。

18. 状态机 v1.3（分支化，补 expired / partial_succeeded）

created → interpreting ─┬─→ needs_clarification ── user answers ──→ interpreting
                        └─→ planned → ir_generated → validated ─┬─→ rejected   # Validator/静态 Policy 拒绝，终态
                                                                └─→ static_risk_evaluated

static_risk_evaluated
  ├─ Low ──→ executing_preflight                  # 快速通道；preflight 内完成动态复评，
  │                                               # 复评升级则转 preview_ready 走确认流程
  └─ Medium / High ──→ preview_ready → dynamic_risk_evaluated ─┬─→ rejected    # 超限，终态
                                                               └─→ waiting_confirmation
                                                                     → confirmed
                                                                     → [waiting_second_confirmation → second_confirmed]  # High
                                                                     → executing_preflight

executing_preflight ─┬─→ stale                    # 复核与 preview 快照不一致，终态
                     └─→ executing → succeeded | partial_succeeded | failed

waiting_confirmation / confirmed / waiting_second_confirmation / second_confirmed
  → expired                                       # 确认超时，终态（v1.3 新增）

任何非终态 → cancelled                             # 用户取消，终态（v1.3：不限阶段）

终态：succeeded / partial_succeeded / failed / rejected / stale / expired / cancelled。终态任务不可复用，需重新 interpret。每次状态迁移写 Audit Log。

20. export_objects 安全规则

  1. 导出路径不得由 LLM 自由指定。
  2. 导出路径必须来自用户选择，或落在 allowed_export_roots。
  3. 禁止相对路径逃逸：../、软链接逃逸、UNC 未授权路径。
  4. 导出格式必须白名单：csv、json、xlsx。

21. 前端卡片

  • Intent Card：展示系统理解的对象、范围、操作。
  • Clarification Card：信息不足时提问。
  • Plan Card：展示执行步骤。
  • Preview Card：展示风险等级、确认等级、命中数量、新值、可撤销、过期时间、警告。
  • Confirm Card：中风险确认；按钮防重复点击。
  • Second Confirm Card：高风险二次确认。
  • Result Card：展示 cad_result_v1（含 partial_succeeded 的失败对象列表）。
  • Error Card：展示 rejected、stale、expired、failed、cancelled（错误码 + 本地化文案）。

22. 验收标准 v1.3

  1. 自然语言不能直接进入执行器。
  2. LLM 不能直接输出 AutoLISP 并立即执行。
  3. 所有执行必须经过 Intent → Task Plan → CAD-IR。
  4. CAD-IR 是执行层唯一业务输入。
  5. CAD-IR 中不得出现 dry_run 或 execute 模式字段；执行模式只能在 AdapterCommand 中表达。
  6. CAD-IR 必须 validator 通过后才能进入 Risk Policy。
  7. Action 白名单与 Risk Policy 不得矛盾。
  8. LLM 写入的风险字段不得作为执行依据；failure_policy_request 同样不得作为执行依据，生效值由 Static Risk Policy 钳制，High 强制 all_or_nothing（需有测试用例）。
  9. Static Risk Policy 在 Preview 前执行；Dynamic Risk Re-evaluation 在 Dry Run 后执行；Low 风险的动态复评在 execute preflight 完成，复评升级则中止快速通道转入确认流程（需有测试用例）。
  10. 动态复评只能升级风险，不能降级。
  11. Low 风险允许 task_id + idempotency_key 直接 execute；Medium/High 缺 token 一律拒绝。
  12. Medium 风险必须 Preview + Confirm；High 风险必须 Preview + Second Confirm，且 second-confirm 必须持有效一级 confirm_token。
  13. 确认必须绑定 task_id + ir_hash + preview_snapshot_hash + document_id + expires_at；preview 重做后旧 token 失效；确认过期任务置 expired（需有测试用例）。
  14. execute 必须幂等，重复提交不能重复改图；终态任务更换 idempotency_key 也不得重新执行。
  15. execute 前必须重新 Dry Run 防 TOCTOU；复核不一致时任务进入 stale，不得执行，不能复用旧确认。
  16. 执行后必须返回 cad_result_v1；状态枚举与状态机统一（succeeded / partial_succeeded / failed）；对象分类与 preview 统一使用 blocked。
  17. AutoLISP 兜底必须使用 command-s；回滚必须是 UNDO _END 后 _.U，禁止 UNDO _BACK；错误消息必须 JSON 转义（需有测试用例：脚本中途抛错后图纸无残留修改、会话历史未被整体回退）。
  18. 状态机必须包含 rejected、stale、expired、cancelled、partial_succeeded；任何非终态可 cancel。
  19. Preview Card 必须显示风险等级、确认等级、新值、可撤销、过期时间。
  20. document_id / drawing_version 必须由插件端 session 注入，不得由 LLM 生成；修订追踪退化必须落 Audit。
  21. export_objects 必须限制导出目录与格式白名单。
  22. Audit Log 必须实现，不得只作为预留。
  23. 旧 immediate execute 路径必须隔离或禁用。

23. 本轮不做

  • 云端 CAD。
  • 语音识别。
  • 多 CAD 平台完整适配。
  • 构件语义识别。
  • 工程量计算。
  • 复杂图纸理解。
  • 批处理任务队列。
  • 多 Agent 协同。

24. 最终口径

不要做：
NL → AgentLite → DSL / AutoLISP → 立即执行

必须做：
NL
→ Intent
→ Task Plan
→ CAD-IR immutable
→ Validator
→ Static Risk Policy（钳制 max_affected_objects / failure_policy）
→ Task Store
→ Preview / Dry Run（Low 跳过）
→ Dynamic Risk Re-evaluation（只升不降）
→ Confirm / Second Confirm when required
→ Execute preflight Dry Run（TOCTOU 复核）
→ AdapterCommand(mode=execute)
→ AutoCAD Executor（undo group + command-s 兜底回滚）
→ cad_result_v1
→ Audit Log
