Agent Implementation Brief · v1.1

AutoCAD 自然语言插件最终架构执行口径

本文件用于指导开发 Agent 直接将 AutoCAD 自然语言插件实现到最终架构。不做过渡版，不保留自然语言直连执行链路。v1.1 修复了 v1.0 的白名单不一致、风险字段信任 LLM、确认链路无防篡改/防重放、Preview 与执行间的 TOCTOU 等问题。

0. v1.1 修订说明（架构审查结论）

v1.0 的总体方向正确：LLM 与执行层之间用 CAD-IR 隔离、强制 Validator / Safety / Preview / Confirm。但审查发现以下问题，v1.1 已在对应章节修正，开发时以本版为准：

v1.0 问题 v1.1 修正
白名单自相矛盾。Safety Checker 表中出现了 Action 白名单之外的动作（modify_text_height、global_replace、scale_objects、modify_xref 等），按 Validator 规则这些动作会被直接拒绝。 统一白名单（见第 6 节）：文本高度、标注样式、图层属性等修改统一归入 set_property 表达；scale_objects 正式纳入白名单（高风险）；global_replace、modify_xref 在 v1 明确禁用，Validator 直接拒绝。Safety 表只引用白名单内动作。
安全字段信任了 LLM。risk_level、requires_confirmation、max_affected_objects 由 LLM 写入 Task Plan / CAD-IR 并被下游采信，LLM 可将删除标为低风险或将上限写成极大值，绕过整套安全机制。 新增"服务端裁决"原则（第 2、7 节）：风险等级与确认要求由 Risk Policy 按 action + selection_scope 在服务端确定性计算并覆盖 IR 中的声明；max_affected_objects 受策略硬上限钳制。LLM 声明的安全字段仅作展示参考。
确认链路无防篡改 / 防重放。"execute 只接受已确认的 CAD-IR"无法落地：客户端可确认温和 IR 后提交篡改过的 IR；双击确认会重复执行。 引入 task_id + ir_hash（第 9 节）：IR 在 interpret 后由服务端持久化并不可变，preview / confirm / execute 全部按 task_id 引用，confirm 产生有有效期的确认令牌，execute 幂等。
Preview 与执行之间存在 TOCTOU。用户可随时在 AutoCAD 中手动改图，Preview 命中 128 个对象不代表执行时仍是 128 个。 execute 前强制重新 Dry Run 并与 Preview 结果比对（第 9、12 节），命中数量偏差超过阈值则中止并要求重新 Preview。
主流程图缺少 Validator 节点，与验收标准中的最终链路不一致。 第 1 节流程图补全 CAD-IR Validator。
缺少澄清流程。模糊输入（如"把圆改大一点"）只能走 Error Card 或被硬猜。 Intent Parser 支持 needs_clarification，前端新增 Clarification Card（第 5.2、14 节）。
缺少任务状态机与部分失败语义。preview / confirm / execute 分接口但无任务生命周期定义；执行中途失败时回滚策略未规定。 新增第 10 节任务状态机；第 12 节规定部分失败语义（默认整组回滚，on_partial_failure 可配）。
Audit Log 被放进"预留"。强调安全的执行系统不应没有执行日志。 最小审计日志（IR + 裁决 + 结果落盘）纳入本轮必须完成范围（第 15 节）。
细节缺口：selector 无文档标识（AutoCAD 多文档环境下目标不明确）；selection_scope 枚举未定义；AutoLISP undo group 在脚本出错时可能不关闭。 selector 增加 document 字段并定义 selection_scope 枚举（第 5.3 节）；AutoLISP Adapter 要求 *error* 处理器兜底关闭 undo group（第 11 节）。

1. 执行模式

进入 Implementation Mode。
本轮不要做临时 MVP，不要保留旧的直连执行链路。旧链路只能作为代码迁移参考，不能继续作为正式执行入口。

必须废弃的现有路径

自然语言 → AgentLite → DSL / AutoLISP → 立即执行 → 结果卡片

必须实现的最终路径

自然语言→ Intent Parser→ Task Planner→ CAD-IR Generator→ CAD-IR Validator→ Safety Checker→ Preview / Confirm→ Execution Adapter→ AutoCAD Executor→ Result Reporter

其中 Intent Parser 在输入模糊时可分支到澄清流程（见 5.2），不允许硬猜后继续。

2. 总体原则

  1. LLM 不直接控制 AutoCAD。
  2. LLM 不直接生成并执行 AutoLISP。
  3. 自然语言不能直接进入执行器。
  4. 所有执行请求必须先转换为 CAD-IR。
  5. CAD-IR 是执行层唯一输入，且自 interpret 落库后不可变，下游一律按 task_id 引用。
  6. Adapter 只能消费 CAD-IR。
  7. 安全裁决只信服务端：风险等级、是否需要确认、影响对象上限，由 Risk Policy 确定性计算，不采信 LLM 在 IR 中的声明。
  8. 中高风险操作必须 Preview / Confirm。
  9. 所有修改类操作必须支持 undo group。
  10. 执行结果必须结构化返回。
  11. 每次裁决与执行必须写入审计日志。

系统边界

层级 职责
LLM 层 理解自然语言、生成 Intent、生成 Task Plan、生成 CAD-IR、必要时发起澄清。其输出一律视为不可信输入。
确定性执行层 校验 CAD-IR、裁决风险、Dry Run、Preview、Confirm、生成 AutoLISP / .NET 命令、执行 CAD 操作、返回结构化结果、记录审计日志。该层不调用 LLM。

4. 废弃旧入口

必须废弃或隔离以下旧路径：

NL → AgentLite → DSL → execute_immediately
NL → AgentLite → AutoLISP → execute_immediately
NL → free_text_command → AutoCAD

硬性要求

  1. 不允许自然语言直接进入执行器。
  2. 不允许 LLM 输出自由 AutoLISP 后立即执行。
  3. 不允许前端点击后直接调用 run_lisp。
  4. 不允许 DSL 绕过 CAD-IR。
  5. 所有旧 DSL 必须迁移为 CAD-IR 或 Adapter 内部实现细节。
如果需要保留旧代码，统一放到 legacy/，默认不可调用，不接入正式 UI，不接入正式执行 API。

5. 正式执行链路

5.1 Natural Language → Intent

用户输入示例：

把 C-ANNO 图层上的所有圆半径改成 5.0

必须先生成 Intent：

{
  "schema": "cad_intent_v1",
  "intent": "modify_objects",
  "confidence": 0.94,
  "target": {
    "space": "model",
    "layer": "C-ANNO",
    "entity_type": "CIRCLE",
    "selection_scope": "layer"
  },
  "operation": {
    "type": "set_property",
    "property": "radius",
    "value": 5.0,
    "unit": "drawing_unit"
  },
  "constraints": {
    "allow_global_scope": false,
    "requires_existing_layer": true
  }
}
  • Intent 必须结构化。
  • Intent 不得包含可执行脚本。
  • Intent 不得包含 AutoLISP 字符串。
  • Intent 不得直接传入 AutoCAD。
  • 用户使用物理单位（如 mm / m）时，Intent Parser 负责按当前图纸单位换算，并在 Intent 中保留原始表述供前端展示核对。

5.2 澄清分支（Clarification）

输入模糊（缺少范围、缺少数值、指代不明，或 confidence 低于阈值）时，Intent Parser 必须返回澄清请求，不允许硬猜后继续：

{
  "schema": "cad_intent_v1",
  "status": "needs_clarification",
  "questions": [
    {
      "field": "operation.value",
      "question": "把圆半径改成多少？请提供数值和单位。"
    }
  ]
}

前端以 Clarification Card 展示，用户补充后重新进入 Intent Parser。澄清不是错误，不走 Error Card。

5.3 Intent → Task Plan

{
  "schema": "cad_task_plan_v1",
  "task_type": "modify_circle_radius",
  "advisory_risk_level": "medium",
  "steps": [
    {
      "id": "select",
      "name": "select_entities",
      "description": "选择模型空间中 C-ANNO 图层上的圆形实体"
    },
    {
      "id": "count",
      "name": "count_entities",
      "description": "统计命中对象数量"
    },
    {
      "id": "validate",
      "name": "validate_entities",
      "description": "检查图层锁定、冻结、关闭状态"
    },
    {
      "id": "modify",
      "name": "set_radius",
      "description": "将圆半径设置为 5.0"
    },
    {
      "id": "report",
      "name": "report_result",
      "description": "返回执行结果"
    }
  ]
}
  • Task Plan 必须可展示给用户。
  • Task Plan 不能直接执行。
  • Task Plan 必须进入 CAD-IR Generator。
  • advisory_risk_level 仅供展示，真实风险由 Risk Policy 裁决（第 7 节）。

5.4 Task Plan → CAD-IR

{
  "schema": "cad_ir_v1",
  "action": "set_property",
  "selector": {
    "document": "active",
    "space": "model",
    "layer": "C-ANNO",
    "entity_type": "CIRCLE",
    "selection_scope": "layer"
  },
  "property": {
    "name": "radius",
    "value": 5.0,
    "unit": "drawing_unit"
  },
  "execution": {
    "undo_group": true,
    "max_affected_objects": 500,
    "on_partial_failure": "rollback_all"
  },
  "advisory_safety": {
    "risk_level": "medium",
    "requires_confirmation": true
  },
  "reporting": {
    "return_object_ids": true,
    "return_failed_objects": true
  }
}
  • CAD-IR 是正式执行层唯一输入。
  • CAD-IR 必须通过 validator。
  • CAD-IR 不能包含自由脚本文本。
  • CAD-IR 不能包含未校验的 LISP 字符串。
  • CAD-IR 必须声明 action、selector、execution、reporting。
  • selector.document：v1 仅支持 active（当前活动文档）。AutoCAD 为多文档环境，执行时必须校验活动文档未切换。
  • selector.selection_scope 枚举：current_selection | window | layer | layer_and_type | global。修改类操作禁止缺省。
  • advisory_safety 是 LLM 的声明，仅供展示；Safety Checker 会另行裁决并以服务端结果为准。
  • execution.max_affected_objects 由 Risk Policy 按 action 钳制到硬上限，IR 声明值只能更小，不能更大。
  • interpret 完成后，CAD-IR 由 Task Store 持久化并生成 task_id 与 ir_hash，此后不可修改（见第 9 节）。

6. CAD-IR Validator

执行前必须校验 CAD-IR。

最小校验规则

  1. schema 必须是 cad_ir_v1。
  2. action 必须属于白名单，且未被标记为 v1 禁用。
  3. selector 必须明确 space / layer / entity_type 或 selection_scope = current_selection。
  4. 修改类操作必须有明确 selector，且 selection_scope 不得缺省。
  5. 禁止无范围全图修改。
  6. property.name 必须属于对应实体类型的允许属性（按实体类型维护属性白名单）。
  7. property.value 类型必须合法，数值必须在该属性的合理区间内（如半径 > 0）。
  8. max_affected_objects 必须存在，且不超过 Risk Policy 对该 action 的硬上限。
  9. undo_group 对修改类操作必须为 true。
  10. IR 中任何字段不得包含可执行脚本或 LISP 片段（字符串字段做模式拦截）。
  11. 校验失败必须返回结构化错误（字段路径 + 错误码），供 Error Card 展示。

Action 白名单（v1 统一版）

所有属性类修改（文本高度、标注样式属性、图层属性等）统一通过 set_property + selector.entity_type + property.name 表达，不再设独立 action，避免白名单膨胀和不一致。

Action 说明 v1 状态
query_objects查询对象启用
count_objects统计数量启用
highlight_objects高亮对象（不改图）启用
export_objects导出对象信息（仅允许写入受控导出目录）启用
set_property设置实体/图层/样式属性（覆盖文本高度、标注样式、图层属性等）启用
move_objects移动对象启用
copy_objects复制对象启用
scale_objects缩放对象启用
delete_objects删除对象启用
explode_blocks炸开块引用启用
purge_layers清理图层启用
modify_block_definition修改块定义启用
global_replace全图替换v1 禁用，Validator 直接拒绝
modify_xref修改外部参照v1 禁用，Validator 直接拒绝

7. Safety Checker

必须实现最终安全策略，不做简化版。

裁决归属：风险等级、是否需要确认/二次确认、max_affected_objects 硬上限，全部由 risk_policy 按 action + selection_scope 在服务端确定性计算。CAD-IR 中 advisory_safety 与裁决不符时，以裁决为准并在 Preview 中向用户提示差异。LLM 无法通过声明降低风险等级或抬高对象上限。
风险等级 操作（均在第 6 节白名单内） 处理方式
Low Risk query_objects、count_objects、highlight_objects、export_objects 可直接执行，但仍必须经过 CAD-IR + Validator。
Medium Risk set_property、move_objects、copy_objects 必须 Dry Run，必须 Preview，必须用户确认。
High Risk scale_objects、delete_objects、explode_blocks、purge_layers、modify_block_definition 必须 Dry Run，必须 Preview，必须二次确认，必须显示影响范围和可撤销状态。

升级规则：selection_scope = global 时，Medium 自动升级为 High。set_property 作用于图层/样式等共享对象（影响面跨实体）时，同样升级为 High。

强制拦截规则

  1. 没有明确范围，禁止修改。
  2. selection_scope = global 时，必须二次确认。
  3. 命中对象数量超过 max_affected_objects（取 IR 声明值与策略硬上限的较小者），禁止直接执行。
  4. 删除、炸开、清理、修改块定义，禁止直接执行。
  5. 图层 locked / frozen / off 时，不得直接修改。
  6. 修改块定义必须显示所有引用实例数量。
  7. 外部参照修改一律阻断（v1 禁用）。
  8. 执行前必须创建 undo group。
  9. 执行失败必须可解释。
  10. 任何校验失败不得进入 Adapter。
  11. 每次裁决（含拦截）写入审计日志。

8. Preview / Dry Run

中高风险操作必须先 Dry Run。Dry Run 不允许改图，只返回影响范围。

实现路径：Dry Run 复用同一条 Adapter → Executor 链路，但 Adapter 只生成只读选择/统计命令（ssget 级别），Executor 以只读模式执行。不为 Dry Run 单独开第二条执行通道，保证 Preview 数字与真实执行使用同一选择逻辑。

返回格式

{
  "schema": "cad_preview_v1",
  "task_id": "task_20260611_0001",
  "ir_hash": "sha256:9f2c…",
  "status": "preview_ready",
  "task_type": "modify_circle_radius",
  "risk_level": "medium",
  "requires_confirmation": true,
  "requires_second_confirmation": false,
  "expires_at": "2026-06-11T08:35:00Z",
  "impact": {
    "matched_count": 128,
    "modifiable_count": 126,
    "blocked_count": 2,
    "space": "model",
    "layer": "C-ANNO",
    "entity_type": "CIRCLE"
  },
  "operation": {
    "action": "set_property",
    "property": "radius",
    "new_value": 5.0
  },
  "warnings": [
    {
      "code": "LAYER_LOCKED_OBJECTS",
      "message": "2 个对象所在图层不可修改"
    }
  ],
  "undo": {
    "available": true,
    "will_create_group": true
  }
}
  • risk_level / requires_* 为 Risk Policy 裁决结果，不是 IR 声明值。
  • Preview 结果带 expires_at，过期后必须重新 Preview 才能确认。

前端展示内容

即将执行：

操作：修改圆半径
图层：C-ANNO
对象类型：CIRCLE
命中对象：128
可修改：126
不可修改：2（图层锁定）
新半径：5.0
风险等级：中
可撤销：是

确认后才会修改图纸。若期间图纸被手动修改，将重新核对影响范围。

9. Confirm 机制

正式执行 API 必须区分 interpret、preview、confirm、execute。不允许一个接口既 preview 又 execute。

接口

POST /cad-agent/interpret   // NL → Intent/Plan/IR，落库生成 task_id + ir_hash
POST /cad-agent/preview     // 按 task_id Dry Run，返回 cad_preview_v1
POST /cad-agent/confirm     // 按 task_id + ir_hash 记录确认，返回 confirm_token
POST /cad-agent/execute     // 按 task_id + confirm_token 执行

接口要求

  1. interpret 只返回 Intent / Task Plan / CAD-IR（或澄清请求），并持久化 IR 快照。IR 自此不可变。
  2. preview 只 Dry Run，不改图。只接受 task_id，不接受 IR 本体。
  3. confirm 校验 ir_hash 与库中快照一致后记录确认状态，签发一次性 confirm_token（默认 5 分钟有效）。高风险操作的二次确认是第二次独立的 confirm 调用。
  4. execute 只接受 task_id + 有效 confirm_token（低风险任务免 token）。不接受自然语言，不接受裸 IR，不接受 LLM 输出。
  5. execute 必须幂等：同一 task_id 重复调用（如双击确认）返回首次执行结果，不重复执行。
  6. execute 在真正改图前重新 Dry Run 一次，命中数量与 Preview 偏差超过阈值（默认 ±0，即任何变化）则中止，状态置为 stale，要求用户重新 Preview。这是对"用户在确认期间手动改图"的防御。

10. 任务状态机 v1.1 新增

每个任务在 Task Store 中维护以下状态，状态迁移只能由确定性执行层触发：

created → interpreted → validated → previewed → confirmed → executing → succeeded
                                  ↘ rejected（validator/safety 拦截）
              ↘ needs_clarification → interpreted（用户补充后）
previewed / confirmed → expired（确认超时）
executing → failed / partially_failed
confirmed → stale（执行前复核发现图纸已变化）
  • 只有 confirmed 状态的任务可进入 Executor（低风险任务允许 validated 直达）。
  • 终态任务（succeeded / failed / rejected / expired / stale）不可复用，需重新 interpret。
  • 每次状态迁移写入审计日志。

11. Execution Adapter

Adapter 总原则

  1. Adapter 只能消费 CAD-IR。
  2. Adapter 不接收自然语言。
  3. Adapter 不接收 Task Plan。
  4. Adapter 不接收 LLM 直接生成的脚本。
  5. Adapter 负责把 CAD-IR 转成 AutoLISP / .NET 命令。
  6. Adapter 生成的命令必须可追踪、可日志化（随 task_id 落审计日志）。
  7. Adapter 对同一 IR 的输出必须确定（同样输入产生同样命令），便于审计与测试。

当前必须实现 AutoLISP Adapter

AutoLISP Adapter 输入：

{
  "schema": "cad_ir_v1",
  "action": "set_property",
  "selector": {
    "document": "active",
    "space": "model",
    "layer": "C-ANNO",
    "entity_type": "CIRCLE",
    "selection_scope": "layer"
  },
  "property": {
    "name": "radius",
    "value": 5.0
  }
}

AutoLISP Adapter 输出（AdapterCommand）：

{
  "schema": "adapter_command_v1",
  "adapter": "autocad_lisp",
  "task_id": "task_20260611_0001",
  "command_type": "lisp",
  "mode": "execute",            // 或 "dry_run"
  "command": "<generated_lisp>",
  "safe_to_execute": true
}
  • generated_lisp 只能由 Adapter 从 CAD-IR 模板化生成，不能来自 LLM。
  • IR 中的字符串值（图层名等）写入 LISP 前必须转义/参数化，防止注入。
  • generated_lisp 必须包裹 undo group，且必须安装 *error* 处理器：脚本中途出错时兜底结束 undo group 并回滚，防止留下半开的 undo 组。
  • generated_lisp 必须返回结构化执行结果（成功/失败对象句柄、计数），供 Result Reporter 解析。
  • mode = dry_run 时只允许生成选择/统计命令，禁止出现任何修改实体的调用。

必须预留 .NET Adapter

本轮不要求完整实现 .NET 执行，但接口必须预留：

autocad_dotnet_adapter.ts

目标：

CAD-IR → .NET Command Request → AutoCAD Transaction → Result

.NET Adapter 后续负责：更可靠的事务控制、对象 ID 回传、图层状态检测、块定义修改、失败对象定位。块定义修改等高风险动作建议优先迁移到 .NET 事务实现。

12. AutoCAD Executor

Executor 只能执行 Adapter 输出。

  1. Executor 不接收自然语言。
  2. Executor 不接收 Intent。
  3. Executor 不接收 Task Plan。
  4. Executor 只接收 AdapterCommand。
  5. Executor 执行前向 Task Store 核对任务状态为 confirmed（或低风险 validated）。
  6. Executor 执行前核对活动文档与 IR 的 selector.document 一致，已切换则中止。
  7. Executor 执行前重新 Dry Run 比对影响范围（见第 9 节第 6 条），不一致则中止并置 stale。
  8. Executor 执行前创建 undo group。
  9. 部分失败语义：按 IR 的 on_partial_failure 处理，默认 rollback_all（整组回滚，结果状态 failed）；显式声明 keep_successful 时保留成功部分，结果状态 partially_failed 并逐对象列出失败原因。
  10. Executor 串行执行任务，同一文档同一时间只允许一个修改类任务在执行。
  11. Executor 执行后收集结构化结果。
  12. Executor 失败时返回错误码和错误原因。

13. Result Reporter

执行完成后返回结构化结果。

{
  "schema": "cad_result_v1",
  "task_id": "task_20260611_0001",
  "status": "success",
  "command": "modify_circle_radius",
  "summary": {
    "matched_count": 128,
    "modified_count": 126,
    "failed_count": 2,
    "skipped_count": 0,
    "layer": "C-ANNO",
    "entity_type": "CIRCLE",
    "property": "radius",
    "new_value": 5.0,
    "elapsed_ms": 420
  },
  "undo": {
    "available": true,
    "group_id": "undo_20260611_001"
  },
  "errors": [
    {
      "object_id": "A23F",
      "reason": "LAYER_LOCKED"
    }
  ]
}
  • status 枚举：success | partially_failed | failed | aborted_stale | rejected。
  • errors[].reason 使用错误码而非自由文本，前端负责本地化展示。
  • 结果连同 task_id、最终 IR、裁决记录一并写入审计日志。

14. 前端最终交互

前端必须从单一结果卡片改为完整任务流卡片。

必须支持的卡片

  1. Intent Card
  2. Plan Card
  3. Clarification Card v1.1 新增
  4. Preview Card
  5. Confirm Card（高风险时为两段式二次确认）
  6. Result Card
  7. Error Card

前端流程

用户输入
→ 显示正在理解
→ 模糊输入：显示 Clarification Card，补充后重新理解
→ 显示 Intent / Plan
→ 生成 CAD-IR（服务端落库，得到 task_id）
→ Validator + Safety Checker（服务端裁决风险）
→ 如果 low risk：允许直接执行
→ 如果 medium risk：显示 Preview + 确认按钮
→ 如果 high risk：显示 Preview + 二次确认
→ 执行（带 confirm_token；按钮防重复点击，confirm 过期则提示重新 Preview）
→ 显示 Result Card / Error Card
不要再只显示“正在初始化几何审计 / 正在识别圆形实体 / 执行成功”。结果必须结构化、可追踪、可解释。Preview 中展示的风险等级必须来自服务端裁决，而非 LLM 声明。

16. 验收标准

  1. 自然语言不能直接进入执行器。
  2. LLM 不能直接输出 AutoLISP 并立即执行。
  3. 所有执行必须经过 Intent → Task Plan → CAD-IR。
  4. CAD-IR 是执行层唯一输入，interpret 后不可变，下游按 task_id 引用。
  5. CAD-IR 必须 validator 通过后才能执行；Safety 表中所有动作均在白名单内。
  6. Safety Checker 必须在执行前运行；风险等级由服务端 Risk Policy 裁决，篡改 IR 中 advisory_safety / max_affected_objects 无法降低安全要求（需有对应测试用例）。
  7. 中风险操作必须 Preview + Confirm。
  8. 高风险操作必须 Preview + 二次确认。
  9. 确认有有效期；execute 幂等，重复提交不会重复改图（需有对应测试用例）。
  10. 确认后 IR 被篡改（ir_hash 不匹配）时，执行必须被拒绝（需有对应测试用例）。
  11. 执行前复核影响范围，Preview 后图纸被手动修改时执行中止并置 stale。
  12. 没有明确范围时，禁止修改图纸。
  13. 删除 / 炸开 / 清理 / 修改块定义禁止直接执行。
  14. 模糊输入走澄清流程，不报错、不硬猜。
  15. 执行前必须创建 undo group；AutoLISP 脚本出错时 undo group 必须被兜底关闭。
  16. 执行后必须返回 cad_result_v1，部分失败按 on_partial_failure 语义处理。
  17. 每次裁决与执行写入审计日志。
  18. 前端必须展示 Plan / Clarification / Preview / Confirm / Result。
  19. 旧的 immediate execute 路径必须隔离或禁用。
  20. AutoLISP 只是 Adapter 输出，不是核心协议。
最终目标状态： 
NL
→ Intent（模糊时 → Clarification → 重新理解）
→ Task Plan
→ CAD-IR（落库，task_id + ir_hash，不可变）
→ Validate
→ Safety（服务端裁决）
→ Preview（Dry Run，带有效期）
→ Confirm（防篡改、防重放）
→ Adapter（确定性生成命令）
→ Executor（执行前复核 + undo group）
→ Result（结构化 + 审计日志）
